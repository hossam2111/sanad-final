🖥️ تشغيل المشروع لوكل من الصفر — كل خطوة
الخطوة 1: تثبيت المتطلبات
# 1. Node.js 24 (لازم 24 تحديداً)
# ادخل على: https://nodejs.org  أو استخدم nvm:
nvm install 24
nvm use 24
# 2. pnpm (مدير الـ packages)
npm install -g pnpm@10
# 3. PostgreSQL 16
# ادخل على: https://www.postgresql.org/download/
# أو على Mac:
brew install postgresql@16
brew services start postgresql@16
الخطوة 2: تحضير المشروع
# 1. نزّل المشروع وافتحه
cd your-project-folder
# 2. اعمل ملف .env في الـ root
echo 'DATABASE_URL=postgresql://postgres:password@localhost:5432/sanad' > .env
# 3. ثبّت كل الـ packages (هيجيب 486 package)
pnpm install
الخطوة 3: قاعدة البيانات
# 1. اعمل الـ database
createdb sanad
# 2. ابني كل الجداول من الـ schema
pnpm --filter @workspace/db run push
# 3. حشو البيانات التجريبية (50 مريض كامل)
pnpm --filter @workspace/scripts run seed
الخطوة 4: تشغيل الـ Servers (3 terminals)
# Terminal 1 — الـ Backend API (port 8080)
PORT=8080 pnpm --filter @workspace/api-server run dev
# Terminal 2 — الـ Frontend (port 5173)
PORT=5173 BASE_PATH=/ pnpm --filter @workspace/sanad run dev
# Terminal 3 (اختياري) — Codegen لو غيّرت الـ OpenAPI spec
pnpm --filter @workspace/api-spec run codegen
الخطوة 5: افتح المشروع
http://localhost:5173
National IDs للتجربة: 1000000001 حتى 1000000050

🔬 التحليل الشامل — فتفوته فتفوته
🏗️ البنية التحتية (Architecture)
workspace/
├── artifacts/
│   ├── sanad/          ← Frontend (React 19 + Vite + Tailwind v4)
│   └── api-server/     ← Backend (Express 5 + TypeScript + Node 24)
├── lib/
│   ├── db/             ← PostgreSQL + Drizzle ORM (9 جداول)
│   ├── api-spec/       ← OpenAPI 3.1 (مصدر الحقيقة)
│   ├── api-zod/        ← Zod schemas (auto-generated)
│   └── api-client-react/ ← TanStack Query hooks (auto-generated)
└── scripts/            ← Seed + Utilities
الفكرة الذكية هنا: الـ OpenAPI spec هي single source of truth — بيتوّلد منها تلقائياً الـ Zod schemas + الـ React Query hooks. يعني لو عدّلت الـ API, الـ frontend بيتحدث أوتوماتيك.

✅ اللي اتعمل فعلاً — بالحجم الحقيقي
1. AI Decision Engine — مش if-else، ده Multi-signal engine حقيقي:

Risk Score:     7 clinical signals → 0-100 score
Urgency:        IMMEDIATE / URGENT / ROUTINE
Time Window:    "Act within 3 hours"
WHY Factors:    كل عامل + وزنه + تأثيره
Digital Twin:   12-month trajectory
Confidence:     89%
Recommendations: مرتبة بالأولوية
2. Emergency Triage — كاملة ومفصلة:

DO_NOT_GIVE → DO_NOT_GIVE → URGENT_REVIEW → MONITOR → ALERT_FAMILY
كل action فيها: icon + color + priority badge + reason
3. Immutable Audit Trail:

WHO (role) → WHAT (action) → WHEN (timestamp) → WHY (reason) → hash
4. SSE Real-time Events — Server-Sent Events لـ:

نتيجة تحليل critical → يوصل للدكتور فوراً
Lab result جديد → broadcast للـ citizen + doctor
5. Auth System:

AuthContext → Protected Routes → Role-based (12 role)
Login page → role selection → portal redirect
