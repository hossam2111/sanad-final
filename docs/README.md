## Reading Order by Audience

### For a Minister / Deputy Minister
1. Executive Summary (2 pages)
2. Government Proposal (formal submission)
3. Use Cases (concrete scenarios)
4. Vision Document (strategic alignment)

### For a CTO / Technical Evaluator
1. Executive Summary (overview)
2. Canonical Narrative (architecture depth)
3. Use Cases (capability proof)
4. Production Readiness Audit (engineering maturity)

### For an Investor / Board Member
1. Executive Summary (value proposition)
2. Vision Document (market + trajectory)
3. Rollout Strategy (execution plan)
4. Canonical Narrative (differentiation depth)

### For a Live Demo
1. Ministry Presentation (slide-by-slide guide)
2. Use Cases (reference during Q&A)
3. DEMO_PLAYBOOK.md (in project root — step-by-step scenario execution)
